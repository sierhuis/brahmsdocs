Overview


Brahms models are written in an Agent-Oriented Language (AOL) that has a well-defined syntax and semantics. The Brahms language is a parsed language. The parser generates an internal object representation for the run-time component. Using this language, a Brahms modeler can create Brahms models. The run-time component - the simulation engine - can execute a Brahms model, also referred to as a simulation. This chapter will teach you how to install and use the various components that form the Brahms system and allow you to write Brahms model and then run them as Brahms simulations.


Installation


Brahms working environment is currently made up of four components:


� Brahms Composer
� Brahms Compiler
� Brahms Simulation Engine (also known as Brahms Virtual Machine)
� Brahms Agent Viewer


These components can be installed by downloading and running the installation/setup files.


Installing Brahms Components


The file setup.exe will start an installation program that will guide you through the steps to install 'PersonalAgent.' PersonalAgent is the core of the Brahms language: you will use its components to write, compile and run your Brahms models.


The installation program will install several executables in a bin directory inside the ../Program Files/PersonalAgent/ directory. One for the Composer (Composer.exe), one for the Compiler (bc.exe), one for the Simulation Engine (Virtual Machine: bvm.exe) and one for the Agent Viewer (agentviewer.exe).


Technical note: the installation program will install batch files in the installation directory to start the Compiler, the Simulation Engine and Brahms Composer with commonly used parameters already set.


Choosing the Brahms model directory


By default, setup.exe will try to install all PersonalAgent files in a folder Brahms\PersonalAgent\ in the c:\Program Files directory. It might be useful to create an additional folder for your projects and their files. You might either use the default 'Models' folder inside the Program Files\Brahms\PersonalAgent\ directory, or create a new folder Brahms in whatever drive you choose to store your projects in (in which case you will have to modify some configuration files, as it will be explained below).

Summary of steps


To summarize, these are the steps necessary to write, compile and run a Brahms simulation:


1. write the Brahms file(s), i.e. files with the .b extension written according to the rules and specifications of the Brahms language.


2. compile them using the Composer or - from the command line - bc.bat; for example, bc.bat c:\Brahms  c:\Brahms\Projects\Atm\AtmModel.b where AtmModel.b is a file that imports the other files in the projects.atm package - more on package in the next chapter - and c:\Brahms is the library path


3. run the Virtual Machine (simulation engine) on the xml files created by the compiler in the same folder where you saved the .b files: bvm projects.Atm.AtmModel


4. parse the resulting history of events .txt file from the AgentViewer, and then open the database file that the parser has just created.

 

Structure of the scenario


Each of the following sections will introduce you to a concept of Brahms modeling: the basic compilation unit; the geography; groups, agents, attributes and relations; facts and beliefs; workframes, thoughtframes and activities; objects and classes; variables; as well as advanced topics like the interaction of many agents, the use of priorities, and the use of random elements.


Each section will adhere to the same structure: a subsection titled 'Introduction' will describe the scope of the section; the 'Task' subsection will present you with the modeling goal to be accomplished by the end of the section; the 'Description' subsection will discuss, more or less formally, the language concepts of interest to the section. The 'Syntax' subsection will link to the complete syntactical details and rules of the concepts being discussed. Finally, the 'Tutorial' subsection will be the area where the 'goal' and the 'description' meet: it will drive you through the actual coding of the Atm scenario. As a general rule, it will be useful to try to code each section's scenario on your own, right after reading the 'Task' and the 'Description', before reading the 'Tutorial' section. Complete model files are also available and linked to from the 'Tutorial' sections.


Expectations and Goals


The reader should have some level of previous programming experience to get the most from this tutorial. Knowledge of object-oriented languages and rule-based languages is preferable, but not essential.


This tutorial will provide a self-contained introduction to the language. It will start with simple concepts and examples and builds up in complexity and difficulty. By the end of this tutorial you should be able to model easy to mildly complex scenarios on your own. The tutorial however will not covered all of the language features. Once you get going, you should refer to the online language specifications for a complete view of the language.


Experienced programmers might move relatively quickly through the initial sections of the Tutorial (Lessons 3 - 8) and spend more time on the final sections, where more complex activities and constructs will be introduced (Lessons 9 -11). Even experienced programmers, however, should try to keep in mind that Brahms is different from object-oriented languages: Brahms is an agent-oriented language with elements of rule-based languages, where activities are not the same things as methods, and rules are not the same as if..then constructs of imperative languages.


One final note before kick-off: this tutorial will not follow the way you are supposed to build a model in Brahms, because you still do not know about Brahms concepts. And remember: there is no substitute for practice to learn a new programming language. As mentioned above, while in the beginning all the details will be described and all the code to implement the cases will be provided, as you will progress in the tutorial less and less details will be provided, and more will be left to your judgmenet. You will have to come out with good solutions to the exercises and challenges proposed in the text. Of course, you will be given examples of code to compare and verify your own solutions.
